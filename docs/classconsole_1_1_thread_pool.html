Detailed Description

线程池执行器类，用于管理和执行并发任务。

该类提供了一个固定大小的线程池，支持提交可调用对象（函数、lambda、绑定表达式等）， 并返回 std::future 对象以获取任务的执行结果。模仿了 Python 的 concurrent.futures.ThreadPoolExecutor 接口。

void console::ThreadPool::abort ( )
inline

粗暴地关闭线程池，用于需要立刻结束一切。

设置关闭标志，不等待已提交的任务，然后分离所有工作线程并退出。 最好不要直接调用，除非必须，调用后线程池将无法再提交新任务。

◆ close()

void console::ThreadPool::close ( )
inline

优雅关闭线程池，等待所有任务完成。

设置关闭标志，等待所有已提交的任务执行完毕，然后等待所有工作线程退出。 该方法会等待任务队列中的所有任务完成后再关闭线程池，调用后线程池将无法再提交新任务。
